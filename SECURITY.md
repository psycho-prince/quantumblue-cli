# Security Policy

## Reporting Vulnerabilities

We take the security of our cryptographic tools and legal-tech infrastructure very seriously. If you have discovered a security vulnerability in QuantumBlue CLI, please report it to us immediately.

### Reporting Process
Please do not report security vulnerabilities through public GitHub issues. Instead, send an email to:

**[INSERT CONTACT EMAIL]**

We will acknowledge receipt of your report within 48 hours and work with you to understand and mitigate the vulnerability.

### Supported Versions
We currently support the following versions:

| Version | Supported |
| :--- | :--- |
| 2.1.0 (Current) | Yes |
| < 2.1.0 | No |

## Responsible Disclosure
We request that you allow us a reasonable time to address the vulnerability before making details public. Thank you for helping us keep our infrastructure secure.
