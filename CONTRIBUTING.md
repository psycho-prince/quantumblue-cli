# Contributing to QuantumBlue CLI

Thank you for your interest in contributing! We welcome contributions to help mature our cryptographic tools and legal-tech infrastructure.

## How to Contribute
1. **Fork the repository.**
2. **Create a feature branch:** `git checkout -b feature/my-new-feature`
3. **Commit your changes:** `git commit -am 'Add some feature'`
4. **Push to the branch:** `git push origin feature/my-new-feature`
5. **Open a Pull Request.**

## Development Standards
- **Code Style:** Follow idiomatic Go standards. Run `go fmt` before submitting.
- **Testing:** New features must include unit tests. Run `go test ./internal/...` to ensure no regressions.
- **Documentation:** Update relevant `.md` documentation if your change affects public APIs or CLI behavior.
- **Compliance:** Ensure all cryptographic changes adhere to the technical specifications documented in `/docs/`.
